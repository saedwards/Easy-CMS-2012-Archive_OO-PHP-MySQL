  • Data\

    Data

    The Data namespace holds all classes and methods that are concerned with reading and writing to the database. Offering a formal way to address structured content from the database using system objects such as items and components.

    • Data\Components\

      Components

      Components are used to create data templates, complex fields and items in the UI. Any type of content creation starts with the use of a component. Components can be added by developers to the \Components\Types\ namespace. and registered with the system.

      • Create

        Used to create a component. Creates an entry in the data_templates table and references this in the components table.

        • __construct($label, $parent = null)

          Required
          A label for the component that's displayed in the editor.
          Optional
          Set a parent for this component to include this as a descendant in a tree.

      • Get

        Gets the component template information from the data_template table.

      • GetTemplate

        Gets the data_template only of the component.

        • byGUID($guid)

          Required
          Global unique identifier needed to reference the right data template.

      • Data\Components\Types\

        Types

        Component types used to create new component items.

        • (IComponentType)

          A Component type implementing IComponentType

          • create()
          • validate()
    • Data\Composition\

      Composition

      Composed items used to manage and create relationships with items with content.

      • Insert

        Creates a child-parent relationship chosen by the user.

        • target(\System\IGUID $obj)

          Specify the target GUID object by passing to this target method
          Required
          A valid item with a system GUID reference must be a descendnt of IGUID interface.

        • add(\System\IGUID $obj, $templateVariable)

          Add items to the target item (has a relationship).

        • getLayoutTemplateVariables()

          Returns the variables listed in the layout template.

    • Data\DataTemplates\

      Data Templates

      Data templates used to create items, which can be made for website visitors or content editors.

      • Create

        Creates a data template used to create items.

        • __construct($label, $parent = null)

          Required
          A label that describes the data template.
          Optional
          Parent to add the data template to a descendant tree.

        • addFieldTypeReference($label, $fieldTypeId)

          Required
          A description label.
          Integer ID of the field type.

      • Get

        Retreives a data template.

        • byGUID($guid)

          Required
          The GUID of the desired data template.

    • Data\Fields\

      Fields

      Data fields hold raw data linked to items in the database. The raw values are stored as their true type in the database.

      • Create

        Will create a new field and field to item reference. The value is saved in a raw data table and linked to an item. The raw value SQL type is determined by the base class inherited from any abstract definition in namespace.

        • __construct(\System\IScalarType $fieldType, $itemGuid)

          Required
          A valid system field type that implements IScalarType.
          An object with a system GUID.

      • CreateNewVersion

        No yet implemented.

      • GetItemFields

        Get all the fields of a GUID object.

        • __construct($itemGuid)

          Required
          A valid registered system guid.

        • getFields()

          Returns all the fields.

      • Data\Fields\Types

        Types

        All system field types created from basic Scalar Types.

        • (IScalarType)

          Objects that extend IScalarType only.

    • Data\Items\

      Items

      Items are content holders representing basic elements of a page. Fields are written into the presentation elements of an item.

      • Create

        Will insert a new item into the table 'items' in the database without any raw data fields or any field references in the 'item_field_catalogue' table.

        • __construct($label, $parent = null)

          Required
          A label that's used for content management.
          Optional
          Parent item of which to become a descendant.

        • addField(\System\IScalarType $field)

          Required
          A field type implementing IScalarType.

      • Edit

        Not yet implemented.

      • Get

        Retreive an item object from the database.

        • byGUID($guid)

          Required
          A valid system GUID string reference.

        • byPath()

          Not yet implemented.

      • GetDescendants

        Retreives a tree of item objects from the database.

        • __construct($guid, $levels = false, \Data\Items\IGetDescendantsCallback $callback = null)

          Required
          A valid system GUID string reference.
          Optional
          The depth of descendants to collect.
          A callback implementing \Data\Items\IGetDescendantsCallback.

        • get()

          Check implementation.

      • GetRoot

        Retreives root items that don't have parents from the database.

        • all()

          Gets all the items in the database that don't have a parent.

        • allWithTemplate()

          Gets all items with a template. (Needs rework with various presentations per item).

        • allWithoutTemplate()

          Gets all items without a template. (Needs rework).

    • Data\Layouts\

      Layouts

      The layout(s) of an item. Custom markup used with placeholder variable strings that are replaced with field values.

      • Create

        Creates a layout reference for a system GUID object.

        • __construct($label, $dir)

          Required
          A label that describes the layout for the content editors.
          The directory of the physical template.

        • getLayout()

          Returns a layout object representing the database entry just created.

      • Get

        Retreives layout details and returns a system layout object.

        • byId($id)

          Required
          The ID of the desired layout

        • all()

          Returns all layout objects.
